基本配置

配置文件

mr_clash使用.yaml格式的配置文件，用户可以通过该文件进行代理设置和规则管理。配置文件通常位于程序安装目录下。

  1. 添加代理信息：在proxies部分添加您选择的代理服务器信息。配置内容包括代理类型、服务器地址、端口、密码等。

  2. 设置流量规则：在rules部分，定义不同流量的走向规则。您可以通过规则控制哪些网站或服务使用代理，哪些则直连。

例如，下面的配置文件展示了如何通过Shadowsocks协议连接代理服务器，并设置规则使得访问google.com的流量走代理。

yaml
proxies:
  - name: "Shadowsocks"
    type: "ss"
    server: "proxy.example.com"
    port: 8388
    password: "your_password"
    cipher: "aes-128-gcm"

rules:
  - DOMAIN-SUFFIX,google.com,Shadowsocks
  - DOMAIN-KEYWORD,facebook,Shadowsocks
  - GEOIP,CN,DIRECT

FAQ

Trojan TLS 伪装域名如何提高抗封锁能力？
通过选择真实可访问域名并使用有效证书，Trojan 流量可以完全模拟 HTTPS 请求。隐藏真实流量特征，降低被封锁和 DPI 检测风险，保证节点长期可用。
Hysteria2 客户端延迟测试值不稳定怎么办？
可通过调整 Hysteria2 的 `recv_window` 与 `send_window` 参数改善传输性能，同时确保服务器带宽充足。若使用移动网络，建议启用 BBR 拥塞控制并降低测速频率以减少波动。
V2Ray 的传输协议选择对性能有何影响？
V2Ray 支持 TCP、WebSocket、gRPC、QUIC 等多种传输协议。不同协议在延迟、抗封锁和吞吐量上差异明显。用户可根据网络环境选择最优协议，以获得稳定且高速的连接体验。
VLESS + XTLS 模式在高并发下有什么优势？
XTLS 模式减少握手次数和加密冗余，提高 CPU 利用率和吞吐量。在高并发场景下能保持连接稳定，适合游戏、流媒体和频繁请求的网络环境，提升整体性能。
Hiddify Next 如何设置流量配额限制？
管理员可在 Hiddify Next 面板为每个用户设置流量上限或带宽限制。超过配额后可自动阻止或降速，保证服务器资源合理分配，防止单用户占用过多带宽。
V2Ray 与 Clash 有什么区别？
V2Ray 是一款代理核心工具，提供底层网络传输和加密支持；而 Clash 是一个多协议客户端，可以整合 V2Ray、Trojan、Shadowsocks 等多种协议。简而言之，V2Ray 负责“传输”，Clash 负责“管理与分流”。
Vmess 节点可以通过 CDN 吗？
可以。Vmess 可通过 WebSocket 与 CDN 结合使用，只需将传输方式设置为 WS 并配置 host 与 path，即可实现流量中转。这样既能隐藏真实服务器，又能提升连接速度。
Nekobox 是否支持跨客户端节点管理？
支持。Nekobox 可以独立管理 Shadowsocks、Clash 等客户端节点，互不干扰。结合策略组，可实现多客户端流量分流和网络优化，提高节点管理效率和使用便利性。
Sing-box 如何兼容 Clash 的配置文件？
Sing-box 支持 Clash 规则格式的解析，用户可直接导入 Clash 的 YAML 配置文件进行使用。虽然部分高级参数需要调整，但大多数分流和策略配置都能自动识别，便于迁移。
Shadowrocket 能否自动更新订阅？
支持。Shadowrocket 可以设置定时更新周期，例如每隔 12 小时或每天自动刷新订阅链接，确保节点列表保持最新状态。这样无需手动更新即可长期稳定使用。
